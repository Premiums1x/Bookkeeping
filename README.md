# 记账小屋（Ledger Vue）

## 项目概述

这是一个面向个人日常收支管理的前端项目，核心目标是让用户以低学习成本完成“记账-筛选-统计-复盘”的闭环。  
项目从单文件页面逐步重构为模块化架构，覆盖完整的账单增删改查流程，并保留轻量、可扩展、可演进的工程特性。

## 技术栈

- **框架与构建**
  - Vue 3（Composition API）
  - Vite 5
- **路由与状态管理**
  - Vue Router 4
  - Pinia
- **网络与数据层**
  - Axios（统一请求实例、拦截器、错误处理）
  - Mock API Adapter（基于 localStorage 的本地接口模拟）
- **UI 与交互**
  - Element Plus
  - 原生 CSS（定制布局与视觉细节）

## 功能与效果

### 1. 账单管理（CRUD）

- 新增账单：类型、金额、分类、备注、日期
- 编辑账单：支持对已有账单全字段修改
- 删除账单：二次确认，防误删
- 查询账单：按月份、类型、日期范围筛选

### 2. 多维筛选与导航

- 月份切换（`YYYY-MM`）
- 日期筛选（今天 / 本月全部 / 近7天 / 指定日期）
- 收支类型切换（全部 / 支出 / 收入）
- 页面路由拆分（账单 / 统计 / 账户）

### 3. 统计与可视化

- 本月支出、收入、预算剩余
- 预算进度条反馈
- 分类支出占比统计
- 按日期分组展示流水与当日净额

## 目录结构

```text
src/
  api/        # axios 实例、业务接口、mock backend
  router/     # 路由配置
  stores/     # Pinia 业务状态
  layouts/    # 页面主布局
  views/      # 账单/统计/账户页面
  components/ # 复用组件（汇总卡片、账单弹窗等）
  utils/      # 日期与金额格式化工具
```

## Mock API 契约

当前通过 axios 访问 `/api/*`，由本地 mock 适配器处理：

- `GET /api/ledger`：获取完整账本状态
- `GET /api/records`：查询记录（支持 month/type/dateFilter/dateValue）
- `POST /api/records`：新增账单
- `PUT /api/records/:id`：修改账单
- `DELETE /api/records/:id`：删除账单
- `PUT /api/budget`：更新预算
- `PUT /api/preferences`：更新筛选与月份偏好

后续切真实后端时，保留 `src/api/ledger.js` 方法签名即可，页面与 Store 无需大改。

## 开发总结与思考

### 1. 先可用，再抽象

先保证功能闭环和交互可用，再推进架构拆分，避免过度设计。

### 2. 状态与视图解耦

将筛选、统计、账单操作下沉到 Store 后，页面层只关注展示和交互，维护复杂度明显下降。

### 3. 接口优先，数据源可替换

通过 API 层统一数据读写，即使当前使用 localStorage 模拟接口，后续迁移真实后端也无需重写页面逻辑。

### 4. 交互一致性与可读性

统一弹窗表单、按钮语义、提示反馈与对齐规则，减少视觉噪声和误操作概率。

## 后续改进方向

- 接入真实后端与账号体系，支持多端同步
- 增强统计分析（趋势图、环比、预算预警）
- 增加测试体系（单元测试、E2E、CI）
- 优化包体积（路由懒加载、手动分包）
- 提升易用性（批量操作、撤销机制、无障碍优化）
